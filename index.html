  • ls -al ~/.ssh - checking for ssh key
  • ssh-keygen -t ed25519 -C "@gmail.com" - generating an ssh key
  • eval "$(ssh-agent -s)" - starting ssh agent
  • sudo -s -H - might need to run for a force of ssh agent check
  • exec SSH-agent bash - to run SSH agent
  • exec SSH-agent zsh - to run SSH agent
  • ssh-add ~/.ssh/id_ed25519 - adding ssh key to your identity
  • clip <> ~/.SSH/id_ed25519 - copying key to clipboard
  • git clone git@github.com:Git-Vdim-Hub/prework-study-guide.git - clones repo to machine in current cd'd folder location
  • cd to the repo after completion
  • git remote -v - checks the url of the repo
  • code . - cd into the directory of a project then use this command to open ms vs code
  • git checkout -b feature/starter-code - Create a feature branch named “starter-code”
  • git status - status of what you are currently updating
  • git commit -m "fixed bug in html main file header"
  • git pull origin main - pull sourcecode from github to insure we have latest commits/changes )if terminal branch is up to date, terminal should say "branch is up to date"
  • **** pulling sourcecode before coding is a good practice to make sure there are no differences between your local copy and the copy on github*****
  • git push origin feature/starter-code - pushes up to your branch and not to main
  • The git command must precede every Git statement.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch.
  • The feature/starter-code is the name of the new branch. We have added the prefix,
  • feature, to indicate that it is a feature. It is good practice to name the branches
  • for the feature that will be developed by them to help indicate the purpose of each branch.
  • git add -A -commits all files in the working branch
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates that we want to add all changes. The staging area informs Git that the changes will be added in the next commit.
  • git commit -m "add starter code"
  • use -m flag to associate a message with our commit. Commit messages should be short descriptive messages of the changes impemented.
  • git pull origin main - check if our local branch is in sync with the base branch in GitHub
  • git push origin feature/starter-code - push our changes to the remote branch
